Program znajduje optymalne rozwiązanie dla podanej ilości płotu oraz nie zawsze wykorzystuje
całość. Program działa w taki sposób że najpierw tworzymy taki wzór gdzie w tym wzorze 
literka Z to ilość metrów dostępnego płotu a więc 2a+b = Z. Dlatego że a*b -> max 
to najpierw wyznaczamy ze wzoru 2a+b = Z przykładowo b a więc. 

2a+b = Z /-2a
b = Z - 2a

następnie pod literke b we wzorze a*b podkładamy to co obliczyliśmy. 

Czyli a*(Z - 2a)

Za - 2a2 

Następnie obliczamy z tego pochodną funkcji 

czyli (Za - 2a2) = Z - 4a

obliczoną pochodną przyrównujemy do zera

Z - 4a = 0 

A więc 
-4a = -Z /:-4
a = Z/4 

b = a*2 czyli inaczej Z/2


Dla reszty optymalnych rozwiązań użyłem odpowiednich obliczeń np dla 15 i innych liczb
ze względu na wymuszoną całkowistość tych liczb.

Hi, it's me Patrick and in this file I will describe my program



The program finds the optimal solution for a given number of fences and does not always use the whole fence. The program works in such a way that first we create a formula where in this formula 
letter Z is the number of meters of available fence, so 2a+b = Z. Because a*b -> max 
we first determine from the formula 2a+b = Z for example b and so. 

2a+b = Z /-2a
b = Z - 2a

then under the letter b in the formula a*b we put what we have calculated. 

So a*(Z - 2a)

Za - 2a2 

Then we calculate from this the derivative of the function 

that is (Za - 2a2) = Z - 4a

The calculated derivative should be compared to zero

Z - 4a = 0 

And so 
-4a = -Z /:-4
a = Z/4 

b = a*2 or in other words Z/2


For the rest of optimal solutions I used appropriate calculations e.g. for 15 and other numbers
due to the forced integrability of these numbers.
